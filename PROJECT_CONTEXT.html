Project shape
03
What is this exactly

A local, single-command Spanish video factory.

One CLI invocation turns a Spanish script into a finished video, captioned and resized for YouTube, TikTok, Reels and Shorts. No web UI. No server. No external API keys required.

Step 01
Script in
Plain text or JSON. Spanish (es-MX preferred).
→
Step 02
Edge-TTS
Free, 45 Spanish neural voices. Outputs audio.mp3 + word timings.
→
Step 03
Remotion Render
React composition → Chrome Headless → H.264 MP4.
→
Step 04
FFmpeg post
loudnorm, multi-platform resize, thumbnail extract.
→
Step 05
Multi-export
1920×1080 (YT), 1080×1920 (TT/Reels), 1080×1080.

Runs ONLY on macOS Apple Silicon. Not deployed anywhere. Not designed to be. The VPS (2 vCPU / 8 GB RAM) is too weak for Remotion + Chrome Headless rendering.

Open work · High priority
06
From scratchpad.md — still valid

Four things that would move the needle.

  • ◉ Integrate faster-whisper into the pipeline. Edge-TTS v7.2.8 only emits SentenceBoundary, so word timings are an even split across each sentence — captions drift visibly on longer sentences. Wiring whisper as a post-TTS pass gives accurate per-word timestamps from the audio itself. The single biggest quality win.
  • ◉ Parallel multi-platform export. The CLI accepts --platforms youtube,tiktok,reels and parses it, but currently only one platform ships per run. FFmpeg jobs are independent — run them with Promise.all instead of sequentially.
  • ◉ Wire @remotion/captions in. The package is installed. Native createTikTokStyleCaptions() would replace the custom Caption.tsx windowed display with Remotion's first-party word animations.
  • ◉ Apply Armando Inteligencia branding. The 5 compositions use generic gradients. Your brand is locked: deep navy #1B3A6E + gold #D4AF37 + Inter / Fraunces. Port the brand config from project 17.

Sharp edges
11

Gotchas that will bite next session if forgotten.

Remotion

  • !staticFile() rejects absolute paths. Audio must be copied to public/ before render.
  • !Folder names must match [a-zA-Z0-9-]. Use Landscape-16x9, never Landscape (16:9).
  • !Don't write .tsx extensions in imports — webpack resolves them. Use ./Root not ./Root.tsx.
  • !All Remotion packages must be exactly the same version. Use npx remotion upgrade.

Edge-TTS + Pipeline

  • !Word-level timing is approximate (sentences distributed evenly). Whisper integration fixes this.
  • !Communicate can only call stream() once. New instance per request.
  • !public/audio.mp3 persists between runs — risk of reusing stale audio if pipeline fails mid-flight.
  • !FFmpeg: -ss before -i seeks by keyframe (fast); after -i seeks exact (slow). Use the right one.
New contender
12
github.com/heygen-com/hyperframes · Apache 2.0 · v0.6.7 (May 15, 2026)

Remotion vs. Hyperframes — the honest comparison.

Incumbent · already installed

Remotion 4.0.443

  • Author in: React / TSX components
  • Build step: Webpack bundler required
  • Engine: Chrome Headless Shell + FFmpeg
  • License: Source-available; free under $1M revenue
  • Distributed: Lambda renderer for fleet rendering
  • Maturity: Battle-tested, 5+ years, large catalog
  • This project: Already wired in, 5 templates, working
vs.
Challenger · 18.4 k stars, active

HeyGen Hyperframes

  • Author in: Plain HTML + data attributes
  • Build step: None — files run as-is
  • Engine: Puppeteer + FFmpeg + GSAP/Anime/Lottie adapters
  • License: Apache 2.0 — no revenue caps, no per-render fees
  • Distributed: Single-machine only (for now)
  • Maturity: v0.6 — young, but 124 releases, 974 commits, used at HeyGen
  • AI-native: Built-in skills framework (npx skills add heygen-com/hyperframes)

Verdict: Hyperframes' draw is the licensing freedom + agent-friendliness. Its weakness is youth. The way to know is to render the same video on both and judge by your eye — installing it next to Remotion costs you only disk space.

The decision
14

Three honest paths from here.

Pick one. I'll execute. None of them block you from picking another later.

Path A · Polish what works

Ship the captions fix and rebrand.

Wire faster-whisper into the pipeline for accurate per-word timing. Run multi-platform exports in parallel. Apply Armando Inteligencia colors + logos to all 5 compositions. Cut a first real branded video this week.

Lowest risk, highest near-term Instagram impact.
Path B · Bake-off

Install Hyperframes side-by-side.

Spin up Hyperframes in a sibling folder, recreate the explainer template in plain HTML, render the same Spanish script through both pipelines, judge by output quality + author DX. Pick a primary engine before sinking another month into either.

Strategic, prevents lock-in. Costs ~half a day.
Path C · Open-ended

Build the first branded video, learn from it.

Skip the bake-off, skip the captions fix. Take Remotion as-is, manually feed in one real Armando Inteligencia script you actually want to publish, push it to completion, see where the friction actually is — then plan from real data.

Fastest signal. Real pain points reveal real priorities.

My recommendation is Path C, then B — make one real video first to surface what's actually painful, then run the bake-off informed by what you learned. But your call.
